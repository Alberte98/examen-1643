Plan para el análisis de seguridad de un si basado en Wordpress

Enunciado:

1. Elabora un plan para el análisis de seguridad de un sistema de información basado en Wordpress.

Paso 1: Recopilación de información

Habrá que llevar a cabo las siguientes indicaciones:

  • Identificar Usuarios y Roles: Identifica a todos los usuarios que interactúan con el sistema y asigna roles y privilegios específicos a cada uno
  • Documentar Políticas de Seguridad: evisa y documenta todas las políticas de seguridad existentes en el sistema, como políticas de contraseñas, políticas de acceso y políticas de retención de datos.
  • Evaluar Procesos de Negocio: i el sistema está vinculado a procesos de negocio, evalúa cómo el sistema contribuye a esos procesos y cómo los procesos impactan en la seguridad del sistema.
  • Inventario de Datos Sensibles Identifica y clasifica los datos sensibles o críticos que se almacenan o procesan en el sistema. Esto puede incluir datos personales, financieros o cualquier otro tipo de información confidencial.
  • Identificar Recursos Críticos Enumera los recursos críticos para el funcionamiento del sistema, como servidores, bases de datos, aplicaciones específicas y cualquier elemento que, si falla, podría tener un impacto significativo en la operación.
  • Flujos de Datos Mapea los flujos de datos a través del sistema para comprender cómo se mueve la información entre los componentes y los usuarios.
  • Identificar Dependencias Externas Identifica cualquier servicio, componente o sistema externo en el que el sistema confía, como servicios de terceros, APIs o servicios en la nube.
  • Historial de Incidentes Revise el historial de incidentes y problemas anteriores relacionados con el sistema para obtener información sobre problemas de seguridad previos.
  • Diagrama de Red: Si el sistema está en una red, crea un diagrama de red que muestre la topología de la red y cómo el sistema se relaciona con otros sistemas y dispositivos.

Paso 2: Identificación de amenazas

La identificación de amenazas implica identificar los eventos o escenarios que pueden poner en peligro la seguridad del sistema.

Los etapas para la deteccion de amenazas son los siguientes

  • Identifica y enumera las amenazas cibernéticas específicas que podrían afectar al sistema. Esto puede incluir:
    1. Ataques de fuerza bruta.
    2. Ataques de inyección SQL en WordPress o phpMyAdmin.
    3. Ataques de denegación de servicio (DDoS) contra el servidor Apache2.
    4. Malware y virus que podrían infectar el sistema.
  • Considera amenazas físicas, como incendios, inundaciones, robos o daños en el hardware del servidor
  • Incluye amenazas relacionadas con errores humanos, como la configuración incorrecta de permisos de archivo, contraseñas débiles o la posibilidad de que un empleado realice acciones maliciosas.
  • Examina las amenazas internas, que pueden ser tanto intencionales como no intencionales
  • Considera amenazas relacionadas con el cumplimiento de normativas y regulaciones, como multas por incumplimiento de la privacidad de datos.
  • Identifica amenazas externas, como la posibilidad de que un atacante externo comprometa la seguridad a través de Internet.
  • Evalúa las amenazas naturales, como terremotos, tormentas o inundaciones, que podrían afectar la ubicación física del sistema.
  • Escenarios de Amenaza: Para cada amenaza identificada, crea escenarios de amenaza específicos que describan cómo podría ocurrir la amenaza y qué impacto tendría en el sistema.
  • Priorización de Amenazas: Clasifica las amenazas identificadas según su probabilidad de ocurrencia y su impacto en el sistema. Esto ayudará a centrar tus esfuerzos en las amenazas más críticas.

Paso 3: Identificación de vulnerabilidades

2.2. Utilizando herramientas de escaneo de seguridad como Wordfence Security para identificar vulnerabilidades conocidas en WordPress y sus componentes (plugins, temas, núcleo).

Wordfence Security (Plugin de WordPress): es un popular plugin de seguridad para WordPress que incluye un escáner de seguridad en tiempo real. Puede escanear tu sitio web en busca de vulnerabilidades conocidas, malware y otros problemas de seguridad. Además, ofrece cortafuegos y otras características de seguridad.

2.3. Verificar si se están utilizando versiones actualizadas de WordPress, plugins y temas.

2.4. Realizar pruebas de penetración para descubrir vulnerabilidades no conocidas.

Para ello se seguiran los siguientes pasos

  • Paso 1: Definir el alcance de las pruebas
  • Paso 2: Obtener el consentimiento y autorización
  • Paso 3: Recopilar información
  • Paso 4: Identificar posibles puntos de entrada
  • Paso 5: Realizar pruebas de seguridad:
    • 5.1. Escaneo de vulnerabilidades automatizado
    • 5.2. Pruebas de inyección de SQL
    • 5.3. Pruebas de cross-site scripting o request forgery (XSS, CSRF)
    • 5.4. Pruebas de autenticación y autorización
  • Paso 6: Documentar resultados
  • Paso 7: Informar y remediar
  • Paso 8: Realizar pruebas de confirmación
  • Paso 9: Revisión continua

Paso 4: Analizarla configuración

4.1. Revisar la configuración de WordPress: garantizar que las mejores prácticas de seguridad se estén siguiendo, como la restricción de acceso a archivos sensibles, la deshabilitación de la edición de archivos desde el panel de administración, entre otros.

4.2. Evaluar la configuración de permisos en archivos y directorios para asegurarse de que los usuarios no autorizados no tengan acceso a información crítica.

Paso 5: Evaluar los plugins y los temas

5.1. Realizar una auditoría de todos los plugins y temas instalados en busca de vulnerabilidades conocidas o problemas de seguridad.

Desde el panel de control de Wordpress, ir al listado de plugins y temas, y verificar de forma infividual cada uno de ellos, y realizar los siguientes pasos:

  • Versión utilizada
  • Escanearo con herramientas de escaneo de seguridad (como WPScan o Nessus)
  • Fiabilidad del plugin o tema
  • Desactivar o eliminar los que generen incompatibilidades
  • Realizar pruebas de funcionalidad
  • Documentar los resultados

Paso 6: Gestionar usuarios y contraseñas

6.1. Revisar la lista de usuarios registrados en WordPress y eliminar cuentas innecesarias o no autorizadas.

Como por ejemplo usuario antiguos que ya no forman parte del cms, cuentas de spam, cuentas duplicadas, e.t.c.

6.2. Fomentar el uso de contraseñas seguras y políticas de contraseña robustas.

Mediante el uso de algun plugin se puede facilitar este proces. Por ejemplo con el plugin SecuPress. Considerar la implementación de autenticación de dos factores (2FA) para los usuarios con privilegios elevados.

Paso 7: Seguridad de la red y el servidor

7.1. Configurar reglas de firewall para limitar el tráfico no deseado.

En el análisis de seguridad, la configuración de reglas de firewall es esencial para controlar el tráfico no deseado en el sistema basado en WordPress. Esto implica establecer reglas de entrada y salida para permitir el tráfico necesario y bloquear el tráfico no autorizado. También se deben considerar reglas específicas para aplicaciones de WordPress y servicios adicionales, junto con la monitorización constante y ajustes según sea necesario para mantener un nivel óptimo de seguridad.

7.2. Mantener el servidor y su software actualizado con parches de seguridad.

7.3. Considerar la implementación de un servicio de monitoreo de seguridad para detectar intrusiones.

Paso 8: Copias de seguridad y recuperación

8.1. Establecer un plan de copias de seguridad regulares que incluya la base de datos y los archivos del sitio web.

8.2. Probar regularmente la restauración de copias de seguridad para asegurarse de que el proceso funcione correctamente.

Paso 9: Educación y concienciación

9.1. Capacitar a los administradores y usuarios sobre buenas prácticas de seguridad.

9.2. Fomentar la identificación de posibles amenazas y cómo informarlas.

Paso 10: Documentación y seguimiento

10.1. Documentar todos los hallazgos, recomendaciones y acciones tomadas durante el análisis de seguridad.

10.2. Establecer un programa de seguimiento y revisión periódica para garantizar que las medidas de seguridad se mantengan actualizadas.

Paso 11: Implementación de medidas correctivas

11.1. Implementar las recomendaciones y medidas correctivas identificadas durante el análisis.

11.2. Realizar pruebas posteriores a la implementación para verificar la efectividad de las soluciones aplicadas.

Paso 12: Monitoreo continuo

12.1. Establecer un sistema de monitoreo continuo para detectar y responder a amenazas en tiempo real.

Una buena implementación poría ser un SIEM (Security Information and Event Management)

Configurar una solución SIEM, como Splunk o Elastic SIEM, que recopile, correlacione y analice registros de seguridad y eventos en tiempo real. Esta herramienta puede alertar sobre actividades sospechosas, como intentos de inicio de sesión fallidos, accesos no autorizados o cambios en archivos críticos. Además, puede automatizar respuestas a incidentes, como el bloqueo de direcciones IP maliciosas o la notificación de alertas a un equipo de seguridad.

12.2. Mantenerse actualizado sobre las nuevas vulnerabilidades y amenazas de seguridad que puedan afectar a WordPress y sus componentes.
