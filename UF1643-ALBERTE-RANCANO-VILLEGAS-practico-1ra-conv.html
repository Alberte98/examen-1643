Plan para el análisis de seguridad de un si basado en Wordpress

Enunciado:

1. Elabora un plan para el análisis de seguridad de un sistema de información basado en Wordpress.

Paso 1: Recopilación de información

1.1. Una única instalación de wordpress, en la versión 6.3.1, montada sobre un Apache2 utilizando como SGBD Phpmyadmin con PHP 8.1.2 y como BBDD Mariadb Ver 15.1.

1.2. Identificación de plugins y temas instalados, así como sus versiones.

  • Tema: Color NewsmagazineVersión en la verión 1.2.7
  • Plugins: Yoast SEO 21.1, Protección Anti-Spam Akismet 5.2, WooCommerce 1.23.10 y Wordfence Security 7.10.3

Paso 2: Evaluación de vulnerabilidades

2.1. Utilizando herramientas de escaneo de seguridad como Wordfence Security para identificar vulnerabilidades conocidas en WordPress y sus componentes (plugins, temas, núcleo).

Wordfence Security (Plugin de WordPress): es un popular plugin de seguridad para WordPress que incluye un escáner de seguridad en tiempo real. Puede escanear tu sitio web en busca de vulnerabilidades conocidas, malware y otros problemas de seguridad. Además, ofrece cortafuegos y otras características de seguridad.

2.2. Verificar si se están utilizando versiones actualizadas de WordPress, plugins y temas.

2.3. Realizar pruebas de penetración para descubrir vulnerabilidades no conocidas.

Para ello se seguiran los siguientes pasos

  • Paso 1: Definir el alcance de las pruebas
  • Paso 2: Obtener el consentimiento y autorización
  • Paso 3: Recopilar información
  • Paso 4: Identificar posibles puntos de entrada
  • Paso 5: Realizar pruebas de seguridad:
    • 5.1. Escaneo de vulnerabilidades automatizado
    • 5.2. Pruebas de inyección de SQL
    • 5.3. Pruebas de cross-site scripting o request forgery (XSS, CSRF)
    • 5.4. Pruebas de autenticación y autorización
  • Paso 6: Documentar resultados:
  • Paso 7: Informar y remediar:
  • Paso 8: Realizar pruebas de confirmación:
  • Paso 9: Revisión continua

Paso 3: Análisis de configuración

3.1. Revisar la configuración de WordPress: garantizar que las mejores prácticas de seguridad se estén siguiendo, como la restricción de acceso a archivos sensibles, la deshabilitación de la edición de archivos desde el panel de administración, entre otros.

3.2. Evaluar la configuración de permisos en archivos y directorios para asegurarse de que los usuarios no autorizados no tengan acceso a información crítica.

Paso 4: Evaluación de plugins y temas

4.1. Realizar una auditoría de todos los plugins y temas instalados en busca de vulnerabilidades conocidas o problemas de seguridad.

Desde el panel de control de Wordpress, ir al listado de plugins y temas, y verificar de forma infividual cada uno de ellos, y realizar los siguientes pasos:

  • Versión utilizada
  • Escanearo con herramientas de escaneo de seguridad (como WPScan o Nessus)
  • Fiabilidad del plugin o tema
  • Desactivar o eliminar los que generen incompatibilidades
  • Realizar pruebas de funcionalidad
  • Documentar los resultados

Paso 5: Gestión de usuarios y contraseñas

5.1. Revisar la lista de usuarios registrados en WordPress y eliminar cuentas innecesarias o no autorizadas.

Como podrían ser usuario antiguos que ya no forman parte del cms, cuentas de spam, cuentas dublicadas, e.t.c.

5.2. Fomentar el uso de contraseñas seguras y políticas de contraseña robustas.

Mediante el uso de algun plugin se puede facilitar este proces. Por ejemplo con el plugin SecuPress

5.3. Considerar la implementación de autenticación de dos factores (2FA) para los usuarios con privilegios elevados.

Paso 6: Seguridad de la red y el servidor 6.1. Configurar reglas de firewall para limitar el tráfico no deseado. 6.2. Mantener el servidor y su software actualizado con parches de seguridad. 6.3. Considerar la implementación de un servicio de monitoreo de seguridad para detectar intrusiones. Paso 7: Copias de seguridad y recuperación 7.1. Establecer un plan de copias de seguridad regulares que incluya la base de datos y los archivos del sitio web. 7.2. Probar regularmente la restauración de copias de seguridad para asegurarse de que el proceso funcione correctamente. Paso 8: Educación y concienciación 8.1. Capacitar a los administradores y usuarios sobre buenas prácticas de seguridad. 8.2. Fomentar la identificación de posibles amenazas y cómo informarlas. Paso 9: Documentación y seguimiento 9.1. Documentar todos los hallazgos, recomendaciones y acciones tomadas durante el análisis de seguridad. 9.2. Establecer un programa de seguimiento y revisión periódica para garantizar que las medidas de seguridad se mantengan actualizadas. Paso 10: Implementación de medidas correctivas 10.1. Implementar las recomendaciones y medidas correctivas identificadas durante el análisis. 10.2. Realizar pruebas posteriores a la implementación para verificar la efectividad de las soluciones aplicadas. Paso 11: Monitoreo continuo 11.1. Establecer un sistema de monitoreo continuo para detectar y responder a amenazas en tiempo real. 11.2. Mantenerse actualizado sobre las nuevas vulnerabilidades y amenazas de seguridad que puedan afectar a WordPress y sus componentes.
